# add-md

提供链接或内容，自动分析后以 ASCII 链路矩阵图集成到 README.md 入口文档树： README.md 做目录入口，每个项目单独 `docs/add-<slug>.md` 展开详情。

## 流程

### 1. 收集输入

- 用户提供链接 → 先用 API / WebFetch 抓取项目信息（名称、描述、作者、语言、Stars、核心功能）
- 用户提供内容描述 → 直接按描述分析
- 多个链接 → 逐个处理
- 若信息不足，追问用户补齐

### 2. 分析并选择图风格

根据项目类型选择 ASCII 图模板：

| 类型 | 适用 | 图风格 |
|------|------|--------|
| 工具/框架 | 有明确处理链 | pipeline 流向 `[A]→[B]→[C]→[D]` |
| 库/包 | 多模块功能 | 4 列模块矩阵 `┌──┬──┬──┬──┐` |
| 平台/服务 | 分层架构 | 上中下层堆叠 |
| Agent/Skill | 触发→推理→行动 | 决策链路 |
| 卡通 | 轻松/创意/社区项目 | 表情角色 `(◕‿◕)` + 手绘框线 `╭──╮` + 可爱情节，讲一个小故事 |
| 其他 | 自定义 | 描述性框图 |

### 3. 创建详细文档

`docs/add-<slug>.md`，包含：
- 项目名称 & 一句话定位
- ASCII 图（完整版）
- 属性表（作者、协议、语言、⭐、核心能力、亮点）
- 适用场景 / cc-undo 跟踪方向
- 原文链接

### 3b. 保存原文副本（可选，仅对文章类链接）

若链接指向的是**文章/博客/专栏页面**（而非 GitHub 项目），且能提取到正文内容：

1. 在 `docs/<slug>-references/` 目录下将原文完整文本保存为 `original-article.txt`
2. 在 `docs/add-<slug>.md` 中添加 "原文全文" 章节，引用该文件路径
3. 加注版权声明：原文版权归原作者所有，此副本仅作本地查阅备份

这确保当原文链接失效或被反爬封锁时，本地仍有可查阅的备份。

### 4. 更新 README.md

若项目已在 README 中 → 提示已存在，跳过。

否则：

**目录** — 若无 `## 目录` 区，在开头标题下插入：
```markdown
## 目录

- [<分类>](#<分类id>)
  - [<项目名>](docs/add-<slug>.md)
```

**分类区** — 若无对应分类区，新建 `## <分类名>`。已有则直接追加。

**条目内容**：
- 简短介绍（1-2 行）
- ASCII 图（缩略版或单行链路）
- 属性表
- 跟踪方向
- `[→ 详细](docs/add-<slug>.md)`

**已有分类参考**：AI / 内容生成、数据采集 / Web、开发工具、框架 / 库、平台 / 服务

### 5. 确认后写入

- 先用消息**展示**完整变更计划
- 等用户确认 → 执行 Write/Edit
- 不自动 commit/push

## 规则

- ASCII 图必须等宽字符，框线用 `─ │ ┌ ┐ └ ┘ ├ ┤ ┬ ┴ ┼`
- 中文字段对齐：全角字符占 2 格，排版时按宽度对齐
- 每个项目独立 .md 文件，放 `docs/` 下
- 已有项目不重复添加
- 分类由 AI 推断，拿不准时问用户
- 原文备份统一放 `docs/<slug>-references/` 目录下，文件名为 `original-article.txt`
- 原文备份仅限文章类链接，不适用于 GitHub 项目页（源码本身即存档）
