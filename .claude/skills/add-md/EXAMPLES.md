# 使用示例

## 示例 1：添加 GitHub 项目

> **用户**：`/add-md https://github.com/D4Vinci/Scrapling`

**AI 执行**：

1. 调用 GitHub API 获取仓库信息 → 描述、语言、Stars、Topics
2. 推断分类 → "数据采集 / Web"
3. 设计图风格 → 模块矩阵（4 列）
4. 创建 `docs/add-scrapling.md` 详细文档
5. 更新 README.md：
   - 若无"数据采集 / Web"分类 → 新建
   - 添加 `Scrapling` 条目 + ASCII 矩阵图 + 链接到详情
6. 展示变更 → 等用户确认 → 写入文件

**产出**：

```
README.md           ← 入口：目录 + 分类区 + 简介 + ASCII + [→ 详细]
docs/
└── add-scrapling.md  ← 详情：完整图 + 属性表 + 跟踪方向
```

## 示例 2：添加非 GitHub 链接

> **用户**：`/add-md https://example.com/some-tool`

**AI 执行**：

1. 用 WebFetch 获取页面内容，提取关键信息
2. 若页面信息不足以推断，追问用户补充
3. 按正常流程处理

## 示例 3：按内容描述添加

> **用户**：`/add-md 我写了一个 CLI 工具，用 Rust 写的，用来批量重命名文件，支持正则和模板`

**AI 执行**：

1. 无链接可抓，按描述提炼关键信息
2. 用 pipeline 流向图（input → regex → template → output）
3. 用户需提供项目名，按正常流程处理
