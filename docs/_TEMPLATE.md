# 文档模板

所有 `docs/add-<slug>.md` 文件遵循以下结构。

## 章节骨架

```
# <项目名>

<一句话定位> —— <补充说明>

\`\`\`
<ASCII 图>
\`\`\`

| 属性 | 说明 |
|------|------|
| 作者 | [<name>](<url>) |
| 语言 | <language> |       ← 单语言项目必填，多语言可选
| ⭐ | <Nk+> |              ← 统一 `k+` 格式（千以下用原始数字）
| 协议 | <license> |         ← 可选，缺失时不填
| 核心能力 | <一句话> |      ← 必填

## 能力         ← 可选，多模块项目用；单功能项目用 ## 适用场景

<能力表或场景列表>

## cc-undo 跟踪方向 ← 必填，最后第二个章节

- <具体、可观察的信号>   ← 每条应指向可验证的缺口
- <...>

[→ GitHub](<url>)     ← 必填，最后一行的结束
```

## 规则

| 要素 | 约定 |
|------|------|
| ASCII 框角 | 标准 `┌┐└┘`（huangting-protocol 等特殊风格例外） |
| 箭头 | pipeline 图用 `───→`（3 破折号 + 箭头），无空格 |
| 缩进 | ASCII 块整体缩进 2 空格 |
| ⭐ 格式 | `163k+`, `51k+`, `6983`（千以下原始数字） |
| 属性行序 | 作者 → 语言 → ⭐ → 协议 → 核心能力（缺失行跳过，不填占位） |
| 跟踪方向 | 聚焦可验证缺口，每条约 10-20 字 |

## README 入口格式

```markdown
### <项目名>

<一句话定位>

[→ GitHub](<url>) · [→ 详细](docs/add-<slug>.md)
```
