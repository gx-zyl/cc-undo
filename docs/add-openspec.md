# OpenSpec

Spec-driven development for AI coding assistants —— 用精确规格驱动 AI 生成靠谱代码。

```
  ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────────┐
  │  编写规格   │───→│  AI 理解  │───→│  代码生成  │───→│   验证闭环    │
  │           │    │          │    │          │    │              │
  │ · 接口定义 │    │ · 语义解析 │    │ · 类型安全 │    │ · 规格校验   │
  │ · 行为描述 │    │ · 上下文   │    │ · 结构匹配 │    │ · 测试对齐   │
  │ · 边界条件 │    │ · 约束推断 │    │ · 模式生成 │    │ · 迭代修正   │
  └──────────┘    └──────────┘    └──────────┘    └──────────────┘
```

## 核心理念

```
❌ "实现一个用户列表页"    → AI 自由发挥，频频偏离预期
✅ "按 OpenSpec 规格"      → AI 按约束精确生成，一次到位
```

- **规范先行**：编码前定义精确接口/行为/边界，规格即测试依据
- **迭代友好**：修改规格 → 重新生成，而非手动改代码
